# 应用更新检查

## 用户故事

- 作为经常打开应用的人，我希望启动时就知道有没有新版本，不用自己去翻下载页。
- 作为想确认版本的人，我希望在设置页有一个“检查更新”按钮，随时查一次。
- 作为看到新版本的人，我希望能先看见更新提示，再决定立刻更新还是稍后再说。

## 为什么需要

现在用户只能自己判断应用是不是旧了。版本一旦落后，修复和新能力就不会自动到手里；用户也没有一个明确入口去确认“我现在是不是最新”。把更新能力放进应用里，可以减少手工查版本的麻烦。

## 怎么解决

应用启动后自动检查一次可用版本；设置页保留手动检查入口。发现新版本时，在设置页显式提示当前版本和可用版本，用户可以选择稍后、下载并重启，或者继续使用当前版本。

## 边界

- 不负责强制用户立刻更新。
- 不做静默强更或后台无感替换。
- 不负责修复安装包以外的系统问题。
- 只解决“知不知道有更新、能不能方便去更新”这件事。

## 当前实现

- 设置页展示当前运行时版本和更新状态。
- 桌面端启动后自动静默检查一次更新。
- 用户可以手动检查更新，并在发现新版本后执行下载并重启，或在 GitHub fallback 场景下直接打开下载页。
- 正式发布继续走 GitHub Release / `latest.json`。
- 本地 updater 验证新增独立脚本链路，不需要为了测试上传 GitHub Release。
