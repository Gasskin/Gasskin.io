# 应用更新脑暴记录

## 1. 想解决什么

让用户能在设置页看到当前应用版本和更新状态，知道自己是不是最新版本；同时保留一个明确的“检查更新”入口，并让应用在启动时自动检查一次更新。

## 2. 已拍板的体验

- 设置页增加一个独立的“关于应用 / 更新”区域。
- 区域里展示版本、构建、平台、Bundle、运行模式这些基础信息。
- 启动时自动检查更新。
- 用户也可以手动点“检查更新”。
- 发现新版本后提示用户下载，并在下载完成后让用户重启安装。
- 不做静默强更，不做后台偷偷安装。

## 3. 推荐的交互

- 默认状态：显示“当前已是最新”或“暂无更新信息”。
- 检查中：按钮显示加载态，避免重复点击。
- 发现新版本：显示新版本号、更新摘要和一个主按钮。
- 失败状态：保留手动重试入口，不阻塞主流程。

## 4. 这次先不做

- 不做强制更新。
- 不做无感静默安装。
- 不做复杂的渠道切换。
- 不在当前轮先决定更新源具体接什么服务。

## 5. 待设计的问题

- 更新信息从哪里来：自建版本清单、GitHub Release，还是别的发布源。
- 下载后是“立刻重启”还是“下次启动安装”。
- 是否要区分稳定版和测试版。
- 更新失败时要不要记录最近一次错误原因。

## 6. 设计输入

这块可以直接进 `cs-feat-design` 了，设计重点会放在页面结构、状态流和更新源接入方式上。
