# Upgrade Does Not Recover Blank Page Issue Report

## 1. 问题现象

装过错误版本的用户，一旦因参考图上传触发过白屏，之后即使重开应用或安装修复版，应用仍持续白屏。

## 2. 复现步骤

1. 安装会触发参考图白屏的旧版本。
2. 在经典工作台选择参考图，触发一次白屏。
3. 关闭应用后重新打开，界面仍然白屏。
4. 直接安装修复版覆盖升级后再次打开，界面仍然白屏。

观察到：应用白屏，且关闭重开或升级到修复版后仍不恢复。

复现频率：一旦触发后稳定复现；旧版本重启不会恢复，升级到修复版后仍持续白屏。

## 3. 期望 vs 实际

**期望行为**：用户安装修复版后，应用应能正常打开，不需要手动清理额外数据。

**实际行为**：只要旧版本曾经触发过一次白屏，之后无论重启旧版还是覆盖安装修复版，应用打开后都仍然是白屏。

## 4. 环境信息

- 涉及模块 / 功能：经典工作台参考图上传后的应用启动恢复
- 相关文件 / 函数：待定
- 运行环境：已安装桌面客户端
- 其他上下文：Windows；首次白屏与参考图上传链路相关；修复版是直接覆盖安装；即使先卸载旧版，只要未清理注册表和数据目录，重新安装后仍会持续白屏

## 5. 严重程度

**P1** — 核心工作台功能受损，已出现过问题的用户即使升级也无法自行恢复。

## 备注

- 该问题从 `2026-06-06-reference-image-upload-blank-page` 拆分而来，单独追踪“坏状态在升级后仍持续存在”的现象。
- 已观察到“仅卸载应用本体”不足以恢复，未清理注册表和数据目录时，重新安装后问题仍然存在。
- 触发问题的错误版本为 `0.0.13`，该版本目前已下线。
